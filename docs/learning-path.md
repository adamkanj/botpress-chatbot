# Two-Week Learning Path

## Week 1

### Day 1-2

- start Botpress locally
- explore the UI without trying to master everything
- identify the main concepts: bot, flow, content, fallback, webchat

### Day 3-4

- create one small learning bot
- add a welcome message
- add a few FAQ-style responses
- test basic paths until you understand how edits show up in chat

### Day 5

- write down what felt intuitive
- write down what felt confusing
- identify where the product feels dated or legacy

## Week 2

### Day 6-8

- expand the bot with one short guided flow
- improve fallback behavior
- update content once or twice to understand the editing cycle

### Day 9-10

- test local webchat/embed behavior
- think about how a separate team could consume the chatbot later
- capture open questions for the real work project

## Success Check

By the end of this learning sandbox, you should be able to:

- run Botpress locally without guessing
- create and update a bot confidently
- explain where flows and content are managed
- show one intentional change in chat output
- explain the likely “our team provides chatbot, their team embeds it” boundary in simple language
